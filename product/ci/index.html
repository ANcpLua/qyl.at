CI telemetry

CI runs are telemetry too. Build phases export as OTLP spans into the same collector as your application traces, so "why was the build slow" is a query, not an archaeology session.

What you get

  • ci_log — CI runs queryable from any MCP client, backed by qyl's own telemetry
  • One span per build phase; failures carry the exception message as span status
  • Dogfooded daily: qyl's own CI reports into qyl

Try it

# from your agent:
ci_log { }  # recent CI runs, phases, failures
