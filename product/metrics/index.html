Metrics

Aggregates that answer agent questions: how many MCP calls, which tools, what did they cost. Token usage and cost come from native execution evidence written by the server itself — not client-side estimates.

What you get

  • fetch_telemetry views — mcp_stats, traces, and more, over any time window
  • Token usage and cost per tool call, from server-owned execution evidence
  • display_mcp_dashboard — an embedded MCP App dashboard for humans

Try it

# from your agent:
fetch_telemetry { "view": "mcp_stats", "hours": 24 }
