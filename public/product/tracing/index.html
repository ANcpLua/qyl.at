Tracing

Standard OTLP in, queryable traces out. The collector ingests /v1/traces into DuckDB; your agent reads them back through MCP tools instead of a dashboard tab.

What you get

  • list_traces / get_trace — query traces and spans from any MCP client
  • display_traces — an embedded trace-explorer MCP App for humans
  • OpenTelemetry semantic conventions, including the GenAI and mcp.* registries
  • Local (qyl up) and hosted (mcp.qyl.at) — same tools, same contract

Try it

qyl up
# point any OTLP exporter at http://127.0.0.1:4318/v1/traces
