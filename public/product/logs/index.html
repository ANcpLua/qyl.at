Logs

Logs live next to the traces they belong to. Instead of grepping a log UI, your agent asks for what it needs.

What you get

  • search_logs — full-text search over collected telemetry from any MCP client
  • list_sessions — session-scoped context to correlate logs with runs
  • Secrets are redacted before anything leaves the server

Try it

qyl up --mcp-stdio npx -y qyl-mcp-server --stdio
# then, from your agent: search_logs { "query": "timeout" }
